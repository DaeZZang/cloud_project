# C4 — CBP-v4 matches default on 2/3 workloads

## Statement

The redesigned CBP-v4 controller — thread-local accumulators with bulk-flush plus a 2-arm counterfactual bandit on `hit_rate` — is **statistically indistinguishable from the static 1/32 default** on TPC-C (Δ = −0.83 %, |Δ| < σ) and Random Read (Δ = −0.23 %, |Δ| < σ). On Skewed YCSB, CBP-v4 underperforms by **−7.4 %** (Δ > σ).

## Evidence

* exp:E3 (n=3 reps each)

## Status

**supported** as stated (2/3 workloads break even). The YCSB regression is a separate failure mode covered by claim:C5.

## Caveats

* "Matches" means "within 1σ"; we cannot claim CBP-v4 *beats* the default on any workload.
* The bandit chooses dramatically different end states across runs (TPC-C run 1: lc=1; run 3: lc=14) — the *throughput* is stable but the controller's internal state is not. This is a stability concern even when the macro-result holds.

## Implication

Adaptive promotion *is* feasible without bookkeeping killing throughput (the round-1 floor is eliminated). But the current hit-rate-based bandit is not the right signal — it doesn't stop at the throughput optimum. A throughput-proxy bandit is the natural next step.
